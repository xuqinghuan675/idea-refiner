# Idea Refiner

[English](./README.en.md) | 中文

> 在你浪费 30 小时写代码之前，让 AI 逼问你 10 分钟。

一个结构化追问 prompt。你的模糊 Side Project 想法进去，MVP 计划出来——或者一句"先别写代码"。

---

## 快速开始

1. 打开 [prompt.md](./prompt.md)，把第一行里的方括号内容替换成你的想法
2. 复制全文，发给 Claude / ChatGPT / DeepSeek
3. 第三轮会问你"当前唯一主目标是什么"——这是整场对话的分叉口
4. 准备好被问 5-10 轮。随时可以说"够了，出计划"

---

## 它能做什么

| | 普通 ChatGPT | Idea Refiner |
|---|---|---|
| 态度 | 你说什么它帮你实现 | 先追问，再判断该鼓励还是劝退 |
| 提问 | 你问，它答 | 主动问，每轮带 E 逃生舱 |
| 逻辑 | 一视同仁 | 按"当前唯一主目标"分支：自用轻追问，赚钱/影响力深追问 |

## 示例对话一：赚钱目标，被劝停

以下是一次示例对话，基于真实使用场景整理。

用户输入：*"给普通人做证件照和票据整理归档的桌面工具"*

---

**Round 0-1 — 动机（开放式）**

AI：用三句话说说，你为什么想做这个？

> 我电脑里证件照、票据、合同散了一堆，每次找都要翻半天。我知道这是小需求但我真的被烦到了。我喜欢做纯本地、不联网的桌面工具。

AI：你现在打开电脑就能为这个想法做什么？

> 我能先把分类规则写出来。再找几个测试图片跑一跑。

---

**Round 2 — 当前唯一主目标**

AI：这个项目对你来说，当前唯一主目标是什么？只能选一个。

A. 自用——我自己用上就算值  
B. 作品集——做完放简历/GitHub，证明能力就算值  
C. 赚钱——有人付费才算值  
D. 开源影响力——有人 star/安装/用起来才算值  
E. 以上都不是，你自己描述

> C ← 选了赚钱

---

**Round 3 — 核心交互**

AI：用户打开之后的第一个动作？

A. 指定文件夹扫描，展示分类结果，确认后归档  
B. 全盘自动扫描，后台托盘通知  
C. 拖入零散文件处理  
D. 设为监视模式，新增文件自动归档  
E. 以上都不是，你自己描述

> A

---

**Round 4 — 分类方式**

AI：图片分类怎么实现？

A. 纯规则（文件名+尺寸+EXIF）——快，零依赖  
B. 本地 AI 模型（ONNX）——能识别内容，增加几百 MB  
C. 云端 API——最准，但违背"不上传"  
D. 纯规则 + OCR 文字提取辅助分类——更准，但要处理 OCR 依赖  
E. 以上都不是，你自己描述

> D

---

**Round 5 — 使用者和付费者**

AI：谁实际使用？谁会付钱？他们是不是同一个人？

A. 我爸妈使用，但我不确定他们会付钱  
B. 上班族自己使用自己付费，但付费意愿没验证  
C. 小商家使用并付费，可能性最高但还没聊过  
D. 说实话，我分不清谁用、谁付钱  
E. 以上都不是，你自己描述

> D

---

**Round 6 — 当前替代方案**

AI：这事现在别人怎么解决？有同类工具吗？

A. 调研过，有类似工具但不聚焦证件/票据  
B. 大概知道有照片管理软件，但没用过  
C. 没查过  
E. 以上都不是，你自己描述

> C

---

**Round 7 — 定价和第一批用户**

AI：第一批付费用户从哪来？你准备怎么收费？

A. 闲鱼/淘宝挂链接，一次性买断  
B. 发 V2EX/少数派文章引流，先免费后收费  
C. 先免费给一批人用，靠反馈判断是否收费  
D. 还没想过  
E. 以上都不是，你自己描述

> D

---

**自我审视**

目标选了赚钱。关键项里至少三项为空：使用者/付费者不清楚、当前替代方案没查、定价和第一批用户没想过。触发硬话。

**AI 输出**

> 我建议你先别写代码。以下是诊断——
>
> - 你选了赚钱，但说不清谁用、谁付钱
> - 你没查过别人现在怎么解决，也不知道竞品或替代方案
> - 你还没有定价假设，也没有第一批用户路径
> - 你自己被这个需求烦到，这点没问题。但"我被烦到"和"有人愿意付钱"是两件事
>
> 你现在可以做两件事：
> 1. 去闲鱼/淘宝/少数派/V2EX 搜"照片整理""证件照归档""票据管理"，看有没有人在卖、多少钱、评论里骂什么
> 2. 发一篇帖子："你电脑里的证件照和票据是怎么整理的？愿意为自动归档工具付多少钱？" 有 5 个具体回复再回来
>
> 做完这些，如果还想做，回来找我。

## 示例对话二：作品集目标，拿到 MVP 计划

用户输入：*"做一个命令行工具，把 Markdown 文件夹转成带目录的静态文档站"*

**已确认信息**

| 维度 | 结论 |
|------|------|
| 当前唯一主目标 | 作品集 |
| 工具使用者 | 写文档的开发者 |
| 作品观看者 | GitHub 访客、招聘方、同行开发者 |
| 技术栈/时间 | Python，每周 5 小时 |
| 技术边界 | 本地 CLI，输出静态 HTML，可部署到 GitHub Pages |
| MVP 边界 | 不做搜索、多语言、插件系统、自定义域名、热重载 |
| 展示路径 | README GIF + live demo + 一篇短文 |

**AI 输出**

> **一句话定义**：给写文档的开发者，把 Markdown 文件夹一键变成带侧边栏导航的静态文档站。
>
> **风险清单**：
> - 技术风险低，核心转换逻辑简单
> - 真正该注意的是别做太重。你做的是作品集，不是 MkDocs 竞品
> - 展示风险比产品风险更大：如果 README 看不出亮点，这个作品就白做了
>
> **技术选型**：Python + Markdown 库 + Jinja2 模板 + 单命令 CLI
>
> **分步实施**：
> 1. 核心转换：md → html + 自动生成目录树 → 验证：自己的项目能跑
> 2. 模板系统：内置 2 套干净主题 → 验证：换主题 1 分钟内完成
> 3. 部署到 GitHub Pages → 验证：别人能访问 live demo
>
> **不要做清单**：搜索、多语言、插件系统、自定义域名、热重载
>
> **展示路径**：
> - GitHub README 首屏放一句话价值 + 30 秒终端录屏 GIF
> - 用它给一个真实项目生成文档站，链接放 README
> - 写一篇短文："我为什么不用 Docusaurus，而写了一个 200 行文档站生成器"
>
> **证明点**：CLI 设计、文件系统处理、模板渲染、文档能力、发布能力
>
> **验证方式**：README 发出去，一周内有 3 个人 star、收藏或提 issue → 值得继续维护。没有 → 作品集目的已经达到，到此为止也不错。

## 核心设计：按当前唯一主目标分支

第三轮强制选择"当前唯一主目标"。次要目标可以记录，但不驱动判断。

| | 自用 | 作品集 | 赚钱 | 开源影响力 |
|---|---|---|---|---|
| 追问深度 | 轻 | 中 | 重 | 重 |
| 强制用户验证 | 否 | 否 | 是 | 是 |
| 强制竞品/替代方案 | 否 | 否 | 是 | 是 |
| 强制分发 | 否 | 展示路径即可 | 是 | 是 |
| 劝退门槛 | 极高 | 中 | 低 | 低 |
| MVP 输出重点 | 砍到只剩自己需要的 | 强化展示 | 验证付费意愿 | 降低 star/试用摩擦 |

如果用户中途目标漂移，prompt 会先重新确认主目标，再继续追问。

**这不是创意杀手。自用项目不会被误杀。**

---

## 适合谁

- 脑子里有模糊 Side Project 想法，说不清楚从哪开始
- 经常"想做个东西"但不确定做到什么程度算 MVP
- 需要人在你动手之前拦住你——但不是一刀切地拦

## 不适合谁

- 已有清晰 PRD 的团队
- 非技术类项目（社区运营、实体服务）
- 只想听"太棒了"的人
